### Architecture Highlights:

**Stateless Design:**
- Nodes don't store entity-specific data
- BTContext provides all runtime information
- Clean separation between tree structure and execution state

**Composability:**
- Fluent API for tree building (`.AddChild()` returns self)
- Mix-and-match nodes to create complex behaviors
- Decorator patterns for behavior modification

**Provider Integration:**
- BTContext caches all coordinator providers
- Fast access to combat stats, health, movement, abilities
- No per-frame lookups

**Performance:**
- Minimal allocations (nodes reused across evaluations)
- Direct provider references (no GetComponent per frame)
- Optional update intervals for BehaviorTreeRunner

### Quick Start:

1. **Add to enemy GameObject:**
   - Attach `BehaviorTreeRunner` component
   - Or use `SimpleEnemyBehaviorTreeTest` for demo

2. **Build tree in code:**
```csharp
var tree = new BehaviorTree("My AI");
var root = new BTSelector("Root");

// Add branches
root.AddChild(combatBranch);
root.AddChild(patrolBranch);

tree.SetRootNode(root);
treeRunner.SetTree(tree);
```

3. **Set target via context:**
```csharp
var context = treeRunner.GetContext();
context.SetTarget(playerTransform);
```
